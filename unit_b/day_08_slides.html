B4 Documentation — Six Required Sections

  1. Title Page with Creative Brief — Goal of the new interaction
  2. User Flow of Existing App — With critique (pros/cons)
  3. Competitive Analysis — Matrix + quadrant positioning chart
  4. User Task Diagrams — Existing flow + modified flow with visual separator
  5. Paper Prototype + Iterations — Show feedback, iterations, reasoning
  6. High-Fidelity Final Prototype — Device mock-ups, annotations, indistinguishable from native

The documentation illuminates the design.

It does not clash with nor overshadow it.

Clear typographic hierarchy  ·  Consistent grids  ·  Cohesive layout

Making Your Documentation Shine

  • Typographic hierarchy — Consistent heading sizes, weights, and spacing to guide the reader
  • Exporting from Figma — Export frames as PNG or PDF at high resolution
  • Device mock-ups — Place final screens into realistic device frames (Figma plugins help)
  • Annotations — Explain why you made a design decision, not just what is on screen
